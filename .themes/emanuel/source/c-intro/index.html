Intro to C

The Anatomy of a C Program

When programming in C, every program you write will have the same outline:
Example 1 
#include <stdio.h>
#include <stdlib.h>

int main (int argc, char* argv[]) {

    // YOUR CODE GOES HERE

    return EXIT_SUCCESS;
}

Note: You can write comments in your program that are ignored when you run it. For one line comments, this is done with a // as above. For multi-line comments, we format the comment

/* Your multi-line
   comment goes
   here */

Style Guide

Style is paramount in writing a readable, understandable program. Group projects and code collaboration are both big parts of professional programming, so well-formatted code is expected and appreciated.

We will teach you C in a clean, readable style, and your personal flair will show up with practice. As long as your code remains nicely formatted, deviations from the styleguide are acceptable.

Structurally speaking, style when programming in C can be the difference between your code working or not working. We will learn more about this in a later module, though you can take a look at the UNSW's C Style Guide if you are interested.

Your first C program

When learning a new programming language, the first thing we do is write a program to display the message "Hello world" on the screen. This will give us an understanding of the basic syntax and structural layout of a program.

Don't worry if it doesn't make sense yet - this is what we are here to learn.
Example 2: helloWorld.c 
#include <stdio.h>
#include <stdlib.h>

int main (int argc, char* argv[]) {

    printf("Hello world!");

    return EXIT_SUCCESS;
}
A lot of this looks strange. Don't worry about it for now - all you have to notice is this:
When we print something out to the screen, we use printf, a function which takes a string and prints it out to stdout.

At the moment, our program is in a language that we can easily understand. However, the C file alone is not something that the computer can understand. To convert a file to machine language, the C file must go through a process called compilation.

The tool we use for compilation is called GCC. We like to have lots of control over the compilation process, so we do the compilation ourselves from the terminal. First, cd to the directory where you've saved the file. In this example, let's say that the C file with our C code in it is called helloWorld.c. To compile our code, we run: 
$ gcc -Wall -Werror helloWorld.c -o hello
Let's break that command down.

We can see that we're asking GCC to do.. something. We're passing two flags: -Wall and -Werror. Usually when we compile we get some warnings about our code, but our program should still run without any problems. However, the warnings indicate that there's something not quite right about what we're telling the computer to do. Hence, we want to avoid warnings. -Wall tells gcc to show all warnings that are found. -Werror tells gcc to treat all warnings as errors, so that our code won't successfully compile if there are any warnings.

After those two flags, we then need to tell the compiler which file we want to compile (helloWorld.c), and finally, the file that we want to output the executable code to (hello). Note that the output file must come directly after -o. If you try to do something like 
$ gcc -Wall -Werror hello -o helloWorld.c
instead, your helloWorld.c file will be overwritten and you'll lose all of the work that you've done. However, something like 
$ gcc -o hello -Werror helloWorld.c -Wall
is fine.

Variables

When programming, we use variables to store information which we use in our program. They are called variables as their values can be changed. In C (unlike some other languages) the type of a variable cannot be changed. We must choose the correct type of variable to represent the data we want to store in it.

Here are some basic types, commonly used in C programs. There are other types, but we tend to prefer the following to their alternatives.

int Integer
char Character
double Double-precision floating point number

Placeholders

We can print out the value of a variable with printf like we used in the "Hello world" example. To do this, we must format a string. This means we must reserve a place in the string for the variables. This reservation is called a placeholder.

Every type of variable has its own placeholder:

Type Placeholder
int %d
char %c
double %lf
Example 3 
#include <stdio.h>
#include <stdlib.h>

int main (int argc, char* argv[]) {

    int myNumber = 5;
    printf("My favourite number is %d\n", myNumber);

    return EXIT_SUCCESS;
}
