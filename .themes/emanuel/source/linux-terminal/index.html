The Linux Terminal

Foreword

Linux is a computer operating system. Linux is a variant of Unix. Indeed, Linux is named after the original developer of Linux; Linus Torvalds.
Mac OS-X, the operating system that your apple laptops are running, is Unix-based. Thus, whenever "Linux" is mentioned, we are really talking about Unix variants. All instructions covered here will work on Mac laptops.

Unix systems are organised in a hierarchy, or in computer science terms, a tree. The root of the tree, or the highest point in the hierarchy, is called the root directory and is represented simply by /.

Folders in the root directory take the form /<folder-name>, such as /home

Sub-folders are named in the same fashion. My "home directory", the folder assigned to my user, is /home/rafi

And now, it is time to start learning how to navigate around the hierarchy!

To do this, we use the terminal, or console.

Note: in examples concerning the terminal, $ represents the prompt, which is the part of the terminal that waits for the next instruction.

pwd

pwd stands for print working directory. The working directory is the directory that you are currently in.

This is a very handy tool to have when you're using files that are spread across different folders on your computer.

cd

cd stands for change directory. This command is used, as its name might suggest, to move between directories. For example, if you are in your home directory (/home/username) and you want to change to the root directory (/), you would enter the command cd / into the terminal.

ls

ls is the list command. It is used to list the files and sub-directories contained in a directory.

Simply entering ls in the terminal will list the files in the current directory. You can provide options (called arguments) to ls. For example, $ ls / will list the contents of the root directory.

$ ls /
bin  boot  cdrom  dev  etc  home  initrd.img  initrd.img.old  lib  lib32  lib64  lost+found  media  mnt  opt  proc  root  run  sbin  srv  sys  tmp  usr  var  vmlinuz
Most of these tools also have flags, which are short arguments that slightly modify the way a tool works. Some useful flags for ls are:

$ ls -a: list all files, including hidden files and directories (which begin with a .).

$ ls -l: list with long listing format.

You can combine these flags. To list all files in long listing format, you use $ ls -la or $ ls -al.

You can find more flags by reading the manual page for ls (type man ls) into the terminal.

cp

cp is the copy command. cp is used to copy files and directories around the filesystem.

For example, to create a copy of a file as a backup, you could run cp helloWorld.c helloWorld.c.bkp

You can also copy entire directories, but doing so requires you to recursively copy the folder's contents. This means copying the folder, and any files or folders inside that folder, and any folders inside those folders, ...
The command to do that would be $ cp -r existing-folder new-folder which uses the -r flag to mean recursive

You must be very careful with the cp tool, as you can unintentionally overwrite files. It is for this reason that we recommend you always back up important files before venturing into unfamiliar tech territory.

mv

Similar to the cp tool is mv, or move. mv is a very powerful tool, but this is not always a good thing.

You can use mv to rename files or folders, by moving them to a new location. For example:
$ mv old-file new-file

However, if there already exists a file or folder named "new-file", it will be replaced by the new file, and you will lose the old file or folder. Always be careful with your mv instructions.

Think of mv as a "cut and paste" command, where cp is "copy and paste".

rm

A truly terrifying tool, rm is the remove command. It is used to delete files or folders.

You can use rm to delete one or more files at a time:
$ rm myFile
$ rm myFile1 myFile2 myFile3
You can even delete entire folders in one command, using the -r flag (recursive):
$ rm -r myFolder

You can also try to force the deletion of a file (you can't always delete a file or directory that doesn't belong to your user). To do this, you use the -f flag.

Warning: the following command deletes as many files on your computer as possible. If you run it on your computer or someone else's computer, you will be kicked out of this course. No second chances.

You can combine the -r and -f flags to forcefullly, recursively delete every file in a directory.
Running $ rm -rf /* will attempt to delete every file in the filesystem, including any mounted storage systems (USB sticks, external hard drives, etc.)

This command is being taught to you only so you know to avoid it.

mkdir

The mkdir command stands for make directory. It is used to create a new directory in the filesystem.

$ mkdir newFolder will create a directory called newFolder inside the current directory. Like other commands, you can specify a location for mkdir to be run, e.g. $ mkdir /home/rafi/newFolder.

An immensely useful flag for mkdir is -p, which produces no error if a folder already exists: 
$ mkdir /home/rafi
mkdir: cannot create directory ‘/home/rafi’: File exists
$ mkdir -p /home/rafi
$
-p also creates the parents of a directory if they do not already exist. For example, you could do:
$ mkdir -p /home/rafi/newFolder/newChild
instead of
$ mkdir /home/rafi/newFolder && mkdir /home/rafi/newFolder/newChild
Here we see a new tool: &&. This is and, and can chain multiple commands together to run one after the other.

rmdir

As might be obvious by now, the rmdir command stands for remove directory. A relatively weak tool, rmdir can only be used to delete completely empty directories. As discussed before, to delete a non-empty directory we must recursively delete it, using rm -r.

Example usage:
$ rmdir aFolder

Special Folders

There are two special folders contained in every folder. These are . and ...

. is the current directory. It is very useful when running programs, as you can run ./executable instead of, for example, /home/rafi/c-programs/helloWorld.

.. is the parent directory. .. comes in handy when you need to quickly reference a directory that is above the working directory:
$ cd .. will change up to the parent of the working directory
$ ../anotherFolder/executable will run the executable file called executable in the folder called "anotherFolder", contained in the working directory's parent.

chmod

Every file and folder in linux has its own set of permissions: permissions for the owner of the file, for the group that the file belongs to, and permissions for everyone else.

There are nine permissions on every file: read, write (change) and execute (run) for the owner of the file, for the group of the file, and for others who are neither the owner nor in the group.

For example, a file with modes rwxr-xr-- has the following permissions:
read write execute
user
group
others
These permissions (called modes) are modified with the chmod command (change mode).

In the example above, running chmod g+w will add write permissions for the file's group, while chmod o-r will remove read permissions for others.
This would leave us with a permission set rwxrwx---, or
read write execute
user
group
others

chown

To change the owner and group of a file, we use the tool chown.

For example, to change the ower of a file to rafi and the group of the file to plugdev, we would run:
$ chown rafi:plugdev filename

To apply the new owner/group to an entire directory, we use the -R (not -r) flag:
$ chown -R rafi:plugdev foldername
